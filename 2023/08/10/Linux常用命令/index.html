Contos防火墙命令

which

主要用来查找系统PATH目录下的可执行文件。
本质就是查找本地已经安装好的可直接执行的命令。
注意：

  1. 不需要关注在哪个目录下执行，which总会查找全局PATH下的可执行文件。
  2. 若可执行文件不在PATH下，也是无法被which查询到。

whereis

whereis用于查找二进制（命令）、源文件、man文件。
通过文件索引数据库而非PATH来查找。（与which不同之处）
同样不需要关注哪个路径下执行。

将本地文件复制到目标机器：
scp 文件名 用户名@目标机器IP:目标机器路径

将目标机器的文件复制到本地
scp 用户名@目标机器IP:目标机器文件名 本地路径 回车后输入密码

查看端口

netstat
netstat -anp
显示所有已开放端口
netstat -tunlp
正在侦听的所有 TCP 或 UDP 端口，包括使用端口和套接字状态的服务
root 或 sudo 用户身份运行命令
-t – 显示 TCP 端口
-u – 显示 UDP 端口
-n – 显示数字地址而不是主机名
-l – 仅显示侦听端口
-p – 显示进程的 PID 和名称
netstat -tnlp | grep :80
查询指定端口通过grep过滤
注：Proto – 套接字使用的协议。Local Address – 进程侦听的 IP 地址和端口号。PID/Program name – PID 和进程名称。

ss
ss 是新的 netstat，命令选项大致相同。它缺少一些 netstat 功能，但暴露了更多的 TCP 状态，而且速度稍快。
ss -tunlp

lsof
lsof 是一个功能强大的命令行实用程序，它提供有关进程打开的文件的信息。
在 Linux 中，一切都是文件，可以将套接字视为写入网络的文件。
lsof -nP -iTCP -sTCP:LISTEN
获取所有侦听 TCP 端口的列表
-n – 不要将端口号转换为端口名称
-p – 不要解析主机名，显示数字地址
-iTCP -sTCP:LISTEN – 仅显示 TCP 协议状态为 LISTEN 的网络文件
sudo lsof -nP -iTCP:3306 -sTCP:LISTEN
查找正在侦听特定端口(3306)的进程

解压压缩

tar主要参数：

  1. -c（create）：

    • 含义： 创建新的 tar 归档文件。
    • 用法： tar -cvf archive.tar file1 file2 directory/

  2. -x（extract）：

    • 含义： 从 tar 归档文件中提取文件。
    • 用法： tar -xvf archive.tar

  3. -t（list）：

    • 含义： 列出 tar 归档文件中的内容。
    • 用法： tar -tvf archive.tar

  4. -f（file）：

    • 含义： 指定归档文件的名称。
    • 用法： tar -cvf archive.tar file1 file2 directory/

  5. -z：

    • 含义： 使用 gzip 进行压缩和解压缩。
    • 用法： tar -cvzf archive.tar.gz file1 file2 directory/

  6. -j：

    • 含义： 使用 bzip2 进行压缩和解压缩。
    • 用法： tar -cvjf archive.tar.bz2 file1 file2 directory/

  7. -J：

    • 含义： 使用 xz 进行压缩和解压缩。
    • 用法： tar -cvJf archive.tar.xz file1 file2 directory/

  8. -C：

    • 含义： 在指定目录中执行操作。
    • 用法： tar -cvzf /path/to/target/archive.tar.gz -C /path/to/source/ file1 file2 directory/

其他参数和选项：

  • -v（verbose）：

    • 含义： 显示详细信息，逐一列出处理的文件。
    • 用法： tar -cvf archive.tar file1 file2 directory/

  • -T：

    • 含义： 从文件中读取文件列表。
    • 用法： tar -cvf archive.tar --files-from=filelist.txt

  • --strip-components：

    • 含义： 在提取文件时忽略路径的前几个组件。
    • 用法： tar -xzf archive.tar.gz --strip-components=1

  • --exclude：

    • 含义： 排除某些文件或目录。
    • 用法： tar -czvf archive.tar.gz --exclude=dir_to_exclude/ source_directory/

  • -A（append）：

    • 含义： 追加文件到已有的 tar 归档文件中。
    • 用法： tar -Avf existing_archive.tar new_file

查询端口的占用情况

  1. netstat命令：

    该命令会列出所有的网络连接，包括监听的端口和相应的进程信息。其中，-t 表示显示TCP连接，-u 表示显示UDP连接，-l 表示显示监听的连接，-p 表示显示进程信息，-n 表示以数字形式显示端口。

  2. ss命令：

    ss 命令也能够提供类似的信息。参数的含义与 netstat 类似，-t 表示显示TCP连接，-u 表示显示UDP连接，-l 表示显示监听的连接，-p 表示显示进程信息，-n 表示以数字形式显示端口。

内存占用情况

  1. free命令：

    该命令显示系统的内存使用情况，包括已用内存、空闲内存、缓冲区和交换空间。-h 参数以人类可读的格式显示数据。

  2. top命令：

    top 命令会实时显示系统的性能数据，包括内存使用情况。在 top 界面中，你可以看到各个进程的内存占用情况，以及系统的总体内存使用情况。

  3. htop命令（如果未安装，可以使用 yum install htop 安装）：

    类似于 top，但提供了更友好的交互式界面，并允许通过箭头键和函数键进行操作。
