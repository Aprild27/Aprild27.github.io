Contos防火墙命令

which

主要用来查找系统PATH目录下的可执行文件。
本质就是查找本地已经安装好的可直接执行的命令。
注意：

  1. 不需要关注在哪个目录下执行，which总会查找全局PATH下的可执行文件。
  2. 若可执行文件不在PATH下，也是无法被which查询到。

whereis

whereis用于查找二进制（命令）、源文件、man文件。
通过文件索引数据库而非PATH来查找。（与which不同之处）
同样不需要关注哪个路径下执行。

将本地文件复制到目标机器：
scp 文件名 用户名@目标机器IP:目标机器路径

将目标机器的文件复制到本地
scp 用户名@目标机器IP:目标机器文件名 本地路径 回车后输入密码

查看端口

netstat
netstat -anp
显示所有已开放端口
netstat -tunlp
正在侦听的所有 TCP 或 UDP 端口，包括使用端口和套接字状态的服务
root 或 sudo 用户身份运行命令
-t – 显示 TCP 端口
-u – 显示 UDP 端口
-n – 显示数字地址而不是主机名
-l – 仅显示侦听端口
-p – 显示进程的 PID 和名称
netstat -tnlp | grep :80
查询指定端口通过grep过滤
注：Proto – 套接字使用的协议。Local Address – 进程侦听的 IP 地址和端口号。PID/Program name – PID 和进程名称。

ss
ss 是新的 netstat，命令选项大致相同。它缺少一些 netstat 功能，但暴露了更多的 TCP 状态，而且速度稍快。
ss -tunlp

lsof
lsof 是一个功能强大的命令行实用程序，它提供有关进程打开的文件的信息。
在 Linux 中，一切都是文件，可以将套接字视为写入网络的文件。
lsof -nP -iTCP -sTCP:LISTEN
获取所有侦听 TCP 端口的列表
-n – 不要将端口号转换为端口名称
-p – 不要解析主机名，显示数字地址
-iTCP -sTCP:LISTEN – 仅显示 TCP 协议状态为 LISTEN 的网络文件
sudo lsof -nP -iTCP:3306 -sTCP:LISTEN
查找正在侦听特定端口(3306)的进程

du -sh *

  1. 日常几个错误记录
    1. IDEA自动生成代码后，主键为long类型，需要手动改为包装类型Long。long为基本类型，当创建对象是不给主键赋值会自动给默认值0l，Long类型默认为null，当我们不给值是可以结合雪花算法或者主键自增进行自动赋值。

HashMap

发表于 2021-09-18 更新于 2022-07-31
本文字数： 944 阅读时长 ≈ 1 分钟

原理

HashMap是由数组和链表组成的，主干是Enery数组，每个Enery包含一个key-vlaue键值对。

链表的存在主要是为了解决哈希冲突，如果多次操作经过哈希算法后落到同一个Entry的节点，那么就会在该节点形成链表。jdk8采用数组+链表+红黑树的数据结构，而jdk7之前采用数组+链表的数据结构。

如果定位到的Enrty节点（数组位置）不含链表，那么查找、添加操作就很快，仅需一次寻址即可。

如果定位到的Entry节点包含链表，那么对应添加操作时间复杂度为**o(1)**；对于查询来说，此时需要遍历链表，然后通过key的equals方法逐一对比查找，出于性能考虑，HashMap种的链表出现越少性能越好。

基于hashing的原理，通过put和get存储和获取对象。当给put()方法传递键值时，先对键做一个hashCode()的计算来得到它在bucket数组中的位置来存储Entry对象。当获取对象时，通过get获取到bucket的位置，再通过键对象的equals()方法找到正确的键值对，然后在返回值对象。

JDK8

数据结构：把原本jdk7中的数组加链表的结构变成了一个数组加链表加红黑树的结构

数据节点：原本的Entry节点也变成了一个Node节点

数据插入方式：1.8数据插入的过程使用的是尾插法：尾插法线程安全吗？
尾插法也不是线程安全的，但是它因为是尾插法，并没有改变原本数据插入的顺序，所以在这不会出现链表循环这么一个过程

JDK7

数据结构：数组加链表

数据节点：他的数据节点是一个Entry节点

数据插入方式：他的数据插入过程是使用的是头插法：头插法会造成什么问题？
他扩容的一个过程，里面有一个resize的方法，它又调用了一个transfer的方法，把里面的Entry进行了一个rehash，这个过程当中可能会造成链表的循环，可能在下一次get的时候出现一个死循环的情况。

扩容机制

如果我们没有设置他的capacity(容量)，他的默认初始化容量是16，负载因子是0.75，扩容的时候他会计算出来扩容的阈值，put的时候会先判断这个size是不是大于这个阈值，如果大于这个阈值了，他就会新创建一个原本两倍大小的这么一个集合，然后将原来的Entry，进行一个resize这么一个过程
